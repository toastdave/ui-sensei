# First Builder Test Plan

## Target

- Archetype: `developer-tool-modern`
- Build spec: `data/prompts/build-specs/developer-tool-modern-ai-release-copilot.json`

## Purpose

Test whether the first-pass research system can guide an original homepage build without collapsing into generic SaaS output or source-site imitation.

## Build Deliverables

- one implementation plan
- one page outline
- one section-to-pattern mapping
- one motion and accessibility note set
- optionally one coded homepage in a future implementation repo

## Review Deliverables

- one critique using `docs/rubrics/build-review-rubric.md`
- one list of missing or weak rules
- one shortlist of pattern/archetype changes based on the critique

## Key Questions

- Does the build spec produce a coherent first-screen value proposition?
- Does the product-proof shell feel believable and original?
- Are the chosen patterns enough to guide sections without excessive improvisation?
- Does the critique produce useful rule feedback rather than vague taste comments?
