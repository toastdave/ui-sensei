# Build Review Rubric

## Purpose

Use this rubric to review UI generated from the research system.

## Scoring

Score each category from 1 to 5.

## Categories

### Archetype Fit

- Does the result feel aligned with the selected archetype?
- Does it use the right pattern family without copying a source site?

### Visual Polish

- Are spacing, hierarchy, surfaces, and states intentional?
- Does the page feel finished rather than generic?

### Clarity

- Is the value proposition easy to understand?
- Is scanning easy above the fold and across sections?

### Hierarchy

- Are page, section, and component priorities obvious?
- Do the strongest elements earn their emphasis?

### Component Quality

- Are buttons, cards, nav, forms, and proof modules coherent?
- Do states and spacing feel systemized?

### Motion Restraint

- Does motion help comprehension?
- Is there unnecessary motion noise or performance cost?

### Accessibility

- Is contrast sufficient?
- Are interactions keyboard and reduced-motion friendly?
- Are semantics and visible focus considered?

### Performance Discipline

- Are heavy effects justified?
- Does the design avoid obvious mobile fragility?

## Review Output Format

Each review should include:

- project name
- selected archetype
- build spec used
- score table
- top strengths
- rule violations
- anti-pattern violations
- recommended next fixes
